# ArcGIS Pro Foundation 2025 (EAPF_2025) Practice Test

Practice exam materials aligned to the **ArcGIS Pro Foundation 2025** exam (EAPF_2025), based on the official Exam Information Guide and sample questions. Single-page HTML; no server or Node required when loading the question bank from a local file.

**Developer:** Kyros

## Contents

| File | Description |
|------|-------------|
| `EAPF2025_EIG_APR2025.pdf` | Official Esri Exam Information Guide (exam structure, domains, skills measured). |
| `exam-spec.json` | Machine-readable exam metadata and domain breakdown (from the EIG). |
| `questions.json` | Question bank: 300 questions mapped to exam domains. Single source of truth for practice content. |
| `practice-test.html` | Practice test UI: loads the bank from `questions.json` (or via file picker when opened as a local file), then draws 75 random questions per exam. |
| `practice-test-standalone.html` | Single-file version with the question bank embedded; works when opened directly (`file://`) with no file picker or server. Generated by `build_standalone.py`. |
| `build_standalone.py` | Script that embeds `questions.json` into the practice test HTML to produce `practice-test-standalone.html`. Run after updating questions. |
| `serve.py` | Optional: run a local HTTP server so `practice-test.html` can load `questions.json` via fetch. Run `python serve.py`, then open the URL it prints. |
| `generate_questions_cursor.py` | Generate questions using Cursor's in-editor model (no API key): `prompt [--count N]` writes a prompt to paste into Cursor; `merge <file> [--count N]` appends the model's JSON output to `questions.json`. |
| `generate_questions.py` | Template-based question generator. Default: 100 questions, start auto-detected from `questions.json`. Run `> new_questions.json 2>&1 && python generate_questions_cursor.py merge new_questions.json` to append. |
| `emit_bulk_questions.py` | Emit many unique stems (intro phrases + templates); e.g. `python emit_bulk_questions.py -n 400 > new_questions.json` then `python generate_questions_cursor.py merge new_questions.json`. |
| `generate_help.py` | Helper: run `python generate_help.py --help` or `-h` for a short usage summary of generation and merge commands. |
| `PRD.md` | Product requirements for this practice test. |

## Exam at a glance (from EIG)

- **Questions:** 75  
- **Duration:** 1 hour 30 minutes  
- **Format:** Multiple choice  
- **Domains and weight:**  
  - Data Management **32%**  
  - Mapping and Visualization **36%**  
  - Layouts and Sharing **12%**  
  - Performing Analysis **20%**

## How to use the practice test

**Option A – Standalone (works when opened directly, no file picker)**  
Open `practice-test-standalone.html` in a browser (e.g. double-click). The question bank is embedded, so it loads immediately with no server and no file picker. If you don’t have this file yet, run: `python build_standalone.py` (requires Python 3); then open the generated `practice-test-standalone.html`.

**Option B – Two-file (fetch or file picker)**  
1. **Open the test:** Open `practice-test.html` in a browser.  
2. **Load the question bank:**  
   - **Served over HTTP (recommended):** Run `python serve.py` in this folder; it starts a local server and opens the test in your browser. The app loads `questions.json` automatically.  
   - **If you opened the file directly** (`file://`): Use “Load from your computer” and choose `questions.json` in this folder.  
3. **Take an exam:** Each run shows **75 questions** drawn at random; question and option order are randomized.  
4. **New exam (random 75)** / **Submit answers** / **Reset answers** as needed.

The question bank includes sample questions from [CertFun’s EAPF_2025 sample questions](https://www.certfun.com/esri/esri-arcgis-pro-foundation-eapf-2025-certification-sample-questions) plus additional items aligned to the EIG domains.

## Adding more questions

**Option 1 – Cursor (no API key)**  

**Automated (recommended):** Run the wizard. It generates the prompt, opens the prompt file, prints what to do in Cursor, then waits for you to save `new_questions.json` and runs merge (and optionally `build_standalone.py`).

```bash
python generate_questions_cursor.py wizard
# Optional: --count 50 | --no-open (don't open the file) | --build (run build_standalone.py after merge)
```

**Manual steps** (what the wizard guides you through):

1. **Generate the prompt**  
   Run: `python generate_questions_cursor.py prompt` (or `wizard`). This creates `prompt_for_cursor_questions.txt`.

2. **Open Cursor Chat**  
   Paste the prompt file contents into Cursor Chat.

3. **Ask Cursor to generate**  
   Send: *"Generate exactly the number of questions requested above. Output only a JSON array of question objects, no markdown code fence and no explanation before or after. Start with [ and end with ]."*

4. **Copy the model output**  
   Copy the JSON array from Cursor’s response (from `[` to `]` only).

5. **Save as `new_questions.json`**  
   Paste into a new file in this folder and save as `new_questions.json`.

6. **Merge into the bank**  
   Run: `python generate_questions_cursor.py merge new_questions.json`  
   This appends the questions to `questions.json` with the correct ids. (If you use **wizard**, merge runs automatically when you press Enter after saving the file.)

7. **Optional**  
   Run `python build_standalone.py` if you use the standalone HTML.

Example commands (after you have `new_questions.json`):

```bash
# Step 1: generate prompt file
python generate_questions_cursor.py prompt

# Step 6: after saving Cursor’s output as new_questions.json, merge
python generate_questions_cursor.py merge new_questions.json
```

More examples:

```bash
# Prompt for 50 questions (default 100)
python generate_questions_cursor.py prompt --count 50

# Prompt for 200 questions
python generate_questions_cursor.py prompt -n 200

# Merge all questions from file
python generate_questions_cursor.py merge new_questions.json

# Merge at most 50 questions from file
python generate_questions_cursor.py merge new_questions.json --count 50
```

**Option 2 – Template-based (no Cursor)**  
Generate 100 questions (default) from **built-in templates only**—no Cursor Chat or AI. You still run `generate_questions_cursor.py merge` to append the file (merge is just file handling, no model). Start id is auto-detected from `questions.json` when `--start` is omitted.

```bash
python generate_questions.py > new_questions.json 2>&1 && python generate_questions_cursor.py merge new_questions.json
```

Use `--count N` for a different number, or `--start 1` for a fresh bank. Then run `python build_standalone.py` if you use the standalone HTML.

**Manual**  
Append items to the `questions` array in `questions.json` with `id`, `domainId`, `text`, `options` (array of `{ key, text }`), and `correctKey`. Then run `build_standalone.py` if needed.
